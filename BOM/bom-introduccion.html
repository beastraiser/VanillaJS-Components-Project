Un modelo, dos enfoques

En muchas ocasiones, cuando se hace referencia al modelo de objetos del cliente, se generaliza con el DOM (modelo de objetos del documento), como si todos los objetos disponibles formaran parte del mismo modelo. Se trata de un error común que no pro- porciona una comprensión completa de cómo funcionan todos los elementos del cliente.
EI DOM no lo es todo, mucho más sabiendo que técnicamente el DOM no es más que una parte del BOM (modelo de objetos del navegador); al fin y al cabo, el navegador es toda la interfaz que tienen los usuarios a mano para usar una aplicación web.
